# 分支权重搜索评估工具 - 需求文档

## 1. 项目背景

### 1.1 现有系统
当前系统在评估阶段使用三个独立的分支进行文本定位（Text-to-Location）任务：
- **Global Branch**: 全局特征分支，准确率 ~78.41%
- **Object Branch**: 物体特征分支，准确率 ~87.54%
- **Relation Branch**: 关系特征分支，准确率 ~87.01%

### 1.2 当前问题
目前三个分支以 **1:1:1** 的固定比例简单相加得到最终相似度分数：
```
final_score = global_score + object_score + relation_score
```

这种固定比例可能不是最优的，因为：
- 不同分支对不同场景的表达能力不同
- 某些分支可能在特定数据分布下表现更好
- 需要通过实验找到最优的权重组合

### 1.3 优化目标
通过网格搜索（Grid Search）找到三个分支的最优权重组合。

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 权重搜索
- 搜索范围：每个分支权重从 0.0 到 1.0
- 搜索步长：0.1
- 总组合数：11 × 11 × 11 = 1331 种组合
- 权重约束：至少有一个分支权重 > 0

#### 2.1.2 评估指标
支持以下评估指标：
| 指标名称 | 描述 |
|---------|------|
| Hit@K | 排名前K个候选中包含正确位置的比例 |
| Close@K | 排名前K个候选中与真实位置距离小于阈值的比例 |

#### 2.1.3 Top-K 配置
支持多种 Top-K 值进行评估：
- Top-1
- Top-3
- Top-5
- Top-10

### 2.2 输出功能

#### 2.2.1 控制台输出
- 实时显示当前搜索进度
- 显示当前最佳权重组合及准确率
- 显示所有组合的完整结果表

#### 2.2.2 结果保存
- **CSV 文件**：保存所有权重组合的结果，便于后续分析
- **JSON 文件**：保存最佳权重组合及对应的准确率
- **日志文件**：保存完整的搜索过程日志

#### 2.2.3 可视化输出
- 生成权重组合热力图（当固定一个分支权重时）
- 显示不同 K 值下的准确率变化曲线

## 4. 评估流程

### 4.1 预处理阶段
1. 加载三个分支的模型权重
2. 创建数据增强/标准化变换
3. 加载数据集和 Cell 数据集

### 4.2 编码阶段
1. 对查询文本进行编码（三个分支分别编码）
2. 对数据库点云进行编码（三个分支分别编码）
3. 缓存编码结果以加速后续搜索

### 4.3 搜索阶段
```
for w_global in [0.0, 0.1, 0.2, ..., 1.0]:
    for w_object in [0.0, 0.1, 0.2, ..., 1.0]:
        for w_relation in [0.0, 0.1, 0.2, ..., 1.0]:
            if w_global + w_object + w_relation == 0:
                continue  # 跳过全零组合
            score = w_global * global_sim + w_object * object_sim + w_relation * relation_sim
            评估并记录结果
```

### 4.4 后处理阶段
1. 排序所有结果
2. 输出 Top-N 最佳权重组合
3. 保存结果文件

## 6. 性能优化

### 6.1 编码缓存
- 编码结果是一次性计算并缓存的
- 权重搜索阶段只进行相似度计算，不涉及模型前向传播
- 大幅减少搜索时间

### 6.2 并行计算
- 使用 NumPy 向量化操作计算相似度
- 避免 Python 循环中的逐元素计算

### 6.3 预估运行时间
| 阶段 | 时间消耗 |
|-----|---------|
| 模型加载 | ~30秒 |
| 数据编码 | ~5分钟 |
| 权重搜索 (1331组合) | ~10秒 |
| **总计** | ~6分钟 |

## 7. 使用场景

### 7.1 典型使用流程
1. 训练完成后获得三个分支的模型权重
2. 运行权重搜索脚本找到最佳权重组合
3. 使用最佳权重组合进行最终评估或部署

### 7.2 进阶使用
- 可以限制搜索范围（如只搜索 0.2-0.8 区间）
- 可以增加步长（如 0.05）进行精细搜索
- 可以单独分析某个分支的重要性

---

## 8. 注意事项

1. **内存占用**：编码阶段会缓存所有编码结果，需要确保有足够内存
2. **权重归一化**：最终分数不需要手动归一化，因为排名不受缩放影响
3. **分支重要性**：通过分析结果可以判断哪些分支更重要
4. **数据一致性**：确保使用与训练时相同的数据划分
